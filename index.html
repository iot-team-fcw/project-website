Pet Interaction Stream

A streaming platform allowing viewers to interact with pets.

This service runs on Twich[1], where a chatbot monitors the stream chat for viewer requests. Viewers are able to submit commands that will move physical devices allowing for animals to play.

Motivation

Motivation

According to a study[2] done by the University of Leeds in the UK, and reported by CNN, it was found that levels of blood pressure, heart rate and anxiety dropped by significant amounts when watching "cute animal videos". With Covid-19 preventing people from living as socially as before, we thought it'd be a good idea to give people a way to interact with animals, without having to do so physically. Thus, the idea of a pet interaction stream was born.

Architecture

At a high-level, this service uses Twitch and cloud services, a python script, a Raspberry Pi and a stepper motor.
Viewers are able to use our Twitch channel to submit chat requests. These requests would be commands, such as 'left' or 'right', to move the corresponding stepper motor. A python script monitors the chat, waiting to direct specific commands to the Raspberry Pi via HTTP requests. The Pi device then communicates with the motors via GPIO commands.

Technical Components

Twitch was chosen as the streaming platform because of its incredible cloud services. Our stream runs on a Rapsberry Pi, yet the stream info and management services are accessible via any browser for the stream host. The Raspberry Pi with the camera broadcasted the stream to our Twitch channel by broadcasting the stream to a specified URL. We used a library[3] to send data, including secret Twitch channel data, to the URL in order to start and maintain stream data tramission. A python script running on a one of the Raspberry Pi interfaced with our channel via Twitch's IRC protocol. The IRC (Instant Relay Chat) protocol allowed for script to interact with stream chats by providing the stream information to their APIs.
Once the script receives commands from the Twitch chat, it polls the most frequent request in the span of a short period of time, and performs the most frequently requested command. The python script running on the Raspberry Pi then invokes a certain GPIO command to move a peripherals. Special serial bus protocols aren't needed because of the simplicity in the commands being executed.

Prototype

Our first prototype had a laser pointer attached to a stepper motor, connected to a Raspberry Pi. Another Raspberry Pi was used to stream a live video feed to our twitch channel. As the python script monitored the chat, it collected commands such as "right", "left", "on" and "off" and performed them within 0.5s of their request. For the initial prototype, no polling of commands was done; instead it was on a first-come-first-serve basis.
